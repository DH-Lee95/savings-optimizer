# 유료 출시 계획

## 현재 구현된 것
- 월 저축 최적화 리포트 가격은 1,490원이다.
- UI에 `출시 첫 일주일 반값 할인` 문구와 기존가 2,980원을 표시한다.
- `/api/checkout`에서 결제 사용권을 만들고, `/api/reports`에서 결제 당시 입력 조건 해시를 검증한 뒤 리포트를 생성한다.
- 같은 리포트 안에서는 `/api/reports/recalculate`로 제외 상품만 바꿔 다시 계산할 수 있다.
- 월 저축액, 기간, 납입 방식, 우대조건을 바꾸면 기존 결제 토큰으로는 리포트를 다시 생성할 수 없다.

## 운영 전에 필요한 것
- 1순위는 검색 등록이나 광고보다 실제 결제 연결이다. 유료 서비스 신뢰와 전환 측정을 위해 `PAYMENT_MODE=mock`을 운영에서 먼저 제거한다.
- Toss Payments 또는 PortOne 중 하나를 골라 결제 승인/취소/환불 API를 붙인다.
- 사용권 저장소를 `data/runtime` 파일에서 managed DB로 옮긴다.
- 결제 성공 웹훅 검증, 주문 금액 검증, 중복 승인 방지, 환불 상태 반영을 서버에서 처리한다.
- 결제 완료 이후에만 리포트를 생성하고, 결제 취소/실패/중복 요청은 리포트를 열지 않는다.
- 이용약관, 개인정보처리방침, 환불정책, 전자상거래 고지 문구를 공개 화면에 추가한다.
- 결제 이벤트와 리포트 생성 이벤트를 서버에 남겨 방문자 대비 결제 전환율을 계산할 수 있게 한다.

## 내일 진행 순서
1. PG사를 결정한다. 빠른 MVP는 Toss Payments 또는 PortOne 중 하나를 우선 검토한다.
2. PG 테스트 상점에서 `clientKey`, `secretKey` 또는 동등한 테스트 키를 발급한다.
3. Render 환경변수에 넣을 이름을 정한다. 예: `PAYMENT_PROVIDER`, `PAYMENT_CLIENT_KEY`, `PAYMENT_SECRET_KEY`, `PAYMENT_MODE`.
4. 서버에 실제 결제 준비 API, 결제 승인 검증 API, 결제 실패/취소 처리를 추가한다.
5. 프론트에서 `1,490원으로 분석하기` 버튼을 실제 결제창으로 연결한다.
6. 결제 성공 후 서버 승인 검증이 끝난 경우에만 `/api/reports`를 호출한다.
7. `checkout_started`, `payment_completed`, `payment_failed`, `report_created`, `recalculate_report` 이벤트를 기록한다.
8. 테스트 결제로 성공/실패/취소/새로고침/뒤로가기/중복 요청을 검증한다.
9. 운영 환경에서 `PAYMENT_MODE=mock`을 끄고 실제 결제 모드로 바꾼다.
10. 결제 연결이 안정화되면 네이버 서치어드바이저, 네이버 애널리틱스, 네이버 광고 전환 추적을 붙인다.

## 홍보 아이디어
- 첫 타깃은 `매달 50만~300만원 저축하는 직장인`으로 좁힌다.
- 메시지는 “내 월 저축액 150만원을 어느 적금에 얼마씩 나눠야 세후로 가장 유리한가”처럼 숫자 중심으로 간다.
- 출시 첫 주는 1,490원 반값 할인과 `추천 상품 이미 사용 중이면 즉시 다음 조합 재계산`을 핵심 차별점으로 보여준다.
- 커뮤니티 홍보는 재테크 카페, 직장인 익명 커뮤니티, 네이버 블로그 계산 예시 글, 숏폼 “150만원 저축 분배 전/후” 콘텐츠가 적합하다.
- 광고 랜딩은 무료 미리보기에서 예상 추가 이익 범위를 보여주고, 상세 조건 반영과 정확한 상품명은 결제 후 노출한다.

## 초기 마케팅 실행안
- 0단계: 전환 추적 전에는 유료 광고를 크게 쓰지 않는다. 방문, 결제 시작, 결제 완료, 리포트 생성이 구분되어야 광고비 회수가 계산된다.
- 1단계: 검색 의도가 강한 키워드부터 테스트한다. 예: `적금 추천`, `적금 금리 비교`, `월 100만원 적금`, `청년 적금 추천`, `파킹통장 금리 비교`.
- 2단계: 네이버 블로그/포스트형 콘텐츠를 만든다. 제목은 `월 150만원 저축, 적금 1개보다 3개로 나누면 얼마 차이날까`처럼 계산 예시형으로 간다.
- 3단계: 커뮤니티에는 직접 판매 문구보다 계산 결과 캡처와 배분 논리를 보여준다. 상품명보다 “조건을 입력하면 분배가 바뀐다”는 사용 이유를 강조한다.
- 4단계: 1,490원은 충동 결제 가능한 가격이므로 CTA는 `내 조건으로 계산하기` 하나로 좁힌다. 할인 문구는 `출시 첫 주 1,490원`으로 짧게 유지한다.
- 5단계: 광고는 하루 1만~2만원의 소액으로 시작하고, 결제완료 CPA가 `1,490원 이하`일 가능성은 낮으므로 초반 목표는 결제보다 클릭률과 결제 시작률 검증으로 둔다.
- 6단계: 결제 전환이 생기면 소재를 세 갈래로 나눠 비교한다. `월 저축액 분배`, `신규가입/급여이체 우대조건`, `청년/지역/특수조건 필터링`.

- 결제창 진입률
- 결제 완료율
- 리포트 생성 성공률

- 결제 1건당 광고비
- 환불/문의 발생률
