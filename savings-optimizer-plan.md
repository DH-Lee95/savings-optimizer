# 최적 저축 포트폴리오 서비스 기획 정리

## 1. 대화 요약

처음에는 스마트폰 사용 시간이 많은 연령, 성별, 계층을 기준으로 수익화 가능한 웹페이지 아이디어를 검토했다. 주요 후보로는 생필품 최저가 비교, 초보 투자자용 정보 허브, AI 도구 큐레이션, 자취템 추천, 행사/팝업 큐레이션, 건강기능식품 비교 등이 있었다.

그중 사용자가 제시한 핵심 아이디어는 **최적 저축 포트폴리오 관리 서비스**다.

서비스의 기본 구상은 다음과 같다.

- 여러 금융사의 예금, 적금, 파킹통장 금리와 가입 조건을 비교한다.
- 사용자의 매월 저축액과 보유 목돈을 입력받는다.
- 사용 목적, 기간, 유동성 필요 여부, 우대조건 가능 여부를 반영한다.
- 어떤 상품에 얼마씩 배분하는 것이 가장 효율적인지 계산한다.
- 결과 페이지에서 공식 가입 페이지로 연결한다.
- 이후 만기 알림, 금리 변동 알림, 재배분 추천 등으로 확장한다.

논의 중 확인한 중요한 판단은 다음과 같다.

- 이 서비스는 매일 쓰는 서비스가 아니라, 목돈 발생, 월급 배분, 예금/적금 만기, 금리 변동 같은 이벤트가 있을 때 쓰는 서비스다.
- 따라서 월구독만으로 수익을 기대하기 어렵다.
- 초기에는 무료 계산기로 유입을 만들고, 1회 리포트, 알림, 저장 기능, 제휴 수익으로 수익화하는 구조가 더 적합하다.
- 본질적으로는 "저축 계산기"에서 시작하지만, 장기적으로는 "개인 돈 관리 이벤트 허브"로 확장해야 한다.

## 2. 핵심 문제

일반 사용자는 다음 문제를 겪는다.

- 은행별 예금, 적금, 파킹통장 금리를 일일이 비교하기 어렵다.
- 우대금리 조건이 복잡해서 실제 받을 수 있는 금리를 판단하기 어렵다.
- 목돈은 예금에 넣어야 할지, 파킹통장에 둬야 할지, 나눠야 할지 결정하기 어렵다.
- 매월 저축액은 적금, 파킹통장, ISA, 연금저축 등 어디에 배분해야 할지 모른다.
- 예금자보호 한도, 만기일, 세후 이자까지 고려하기 번거롭다.
- 금융상품 추천 글은 많지만, "내 금액 기준"으로 계산해주는 곳은 적다.

이 서비스는 이 문제를 **사용자 금액 기준의 자동 배분 계산**으로 해결한다.

## 3. 핵심 가치 제안

서비스 한 문장 설명:

> 내 월 저축액과 목돈을 입력하면, 예금, 적금, 파킹통장에 얼마씩 넣는 것이 가장 효율적인지 계산해주는 저축 최적화 웹페이지.

사용자에게 보여줄 수 있는 메시지:

- 내 돈, 은행에 그냥 두면 얼마나 손해일까?
- 월급 300만 원 기준 최적 저축 비율 계산기
- 목돈 1,000만 원, 지금 어디 넣어야 가장 이득일까?
- 예금, 적금, 파킹통장 자동 배분기
- 세후 이자 기준으로 가장 효율적인 저축 조합 찾기

## 4. 초기 타깃 사용자

초기 타깃은 다음 사용자다.

- 월급을 받기 시작한 20대, 30대 직장인
- 목돈 500만 원~5,000만 원을 굴릴 방법을 찾는 사람
- 전세, 결혼, 자동차, 여행, 유학 등 목적자금을 모으는 사람
- 예금/적금 만기가 다가온 사람
- 파킹통장, 예금, 적금 차이를 잘 모르는 금융 초보자
- 고위험 투자보다 안정적인 저축 최적화를 원하는 사람

초기에는 고액 자산가보다 **금융 초보자, 사회초년생, 안정지향 저축자**를 대상으로 하는 것이 좋다.

## 5. MVP 기능

처음 만들 MVP는 복잡한 금융앱이 아니라, 빠르게 관심을 끌 수 있는 계산기형 웹페이지로 잡는다.

필수 기능:

- 사용자 입력 폼
- 상품 데이터 테이블
- 최적 배분 계산
- 결과 카드
- 예상 세전/세후 이자 표시
- 현재 방식 대비 추가 이익 표시
- 공식 가입 링크 연결
- 이메일 또는 카카오 알림 신청 폼

초기 입력값:

- 매월 저축 가능액
- 현재 보유 목돈
- 돈을 사용할 예정 시점
- 중도 인출 필요 여부
- 선호 방식: 최고이자, 안전성, 간편가입, 예금자보호 분산
- 가능한 우대조건: 급여이체, 카드실적, 첫거래, 자동이체 등

초기 출력값:

- 목돈 배분안
- 매월 저축액 배분안
- 상품별 추천 금액
- 예상 세전 이자
- 예상 세후 이자
- 일반 입출금통장 대비 추가 이익
- 가입 순서
- 상품별 가입 링크

예시 결과:

> 월 50만 원 저축 + 목돈 1,000만 원, 1년 뒤 사용할 예정  
> 추천: 파킹통장 200만 원, 12개월 예금 800만 원, 월 적금 50만 원  
> 예상 세후 이자: 약 XX원  
> 그냥 입출금통장에 둘 때보다 약 XX원 이득

## 6. 데이터 파이프라인

### 6.1 수집할 데이터

상품별로 다음 데이터를 수집한다.

- 금융사명
- 상품명
- 상품 유형: 예금, 적금, 파킹통장, CMA 등
- 기본금리
- 최고금리
- 우대금리 조건
- 가입 기간
- 최소 가입금액
- 최대 가입금액
- 월 납입 한도
- 중도해지 조건
- 이자 지급 방식
- 세금 적용 방식
- 예금자보호 여부
- 가입 가능 채널: 앱, 웹, 영업점
- 공식 가입 링크
- 데이터 업데이트 날짜

### 6.2 초기 수집 방식

초기에는 완전 자동 크롤링보다 **반자동 데이터 관리**가 좋다.

이유:

- 금융상품 조건은 예외가 많다.
- 우대금리 조건이 복잡하다.
- 크롤링 구조가 자주 바뀔 수 있다.
- 잘못된 금리 노출은 신뢰와 법적 리스크가 크다.

추천 방식:

- 금융감독원 금융상품 한눈에 데이터 활용
- 주요 은행, 인터넷은행, 저축은행 상품은 수동 보정
- 초기에는 Google Sheets 또는 간단한 DB로 관리
- 하루 1회 또는 주 3회 업데이트
- 페이지에 업데이트 날짜를 명확히 표시

### 6.3 실제 데이터 수집/분석 단계

이 단계는 결제/회원 서버 전환보다 먼저 진행한다. 유료 리포트의 신뢰도는 결제 기능보다 상품 데이터 정확도에 더 크게 좌우되기 때문이다.

1차 구현 범위:

- 네이버페이 예금/적금/파킹통장 페이지를 기본 수집 소스로 사용한다.
- 네이버 페이지의 필터 코드와 상품 목록 응답을 합쳐 raw product로 변환한다.
- 상품 상세 페이지까지 들어가 스크롤 하단의 `금리 안내` 영역을 읽고 우대금리 세부 조건 원문을 저장한다.
- 매일 모든 상세 페이지를 다시 읽지 않고, 전날 목록 요약과 달라진 상품, 신규 상품, 캐시에 없는 상품만 상세 조건을 재수집한다.
- 전날에는 있었지만 오늘 목록에서 사라진 상품은 삭제하지 않고 `removed` 상태로 표시해 검수 대상으로 올린다.
- 같은 상품의 6개월, 12개월, 24개월 조건은 별도 후보로 취급한다.
- 신규 상품과 금리/한도/우대조건 변경 상품은 `pending`으로 등록한다.
- 운영자가 공식 은행 페이지와 우대조건을 확인한 뒤 승인해야 계산에 반영한다.
- 승인 시 공식 가입 링크를 반드시 넣는다.

분석에 반영할 정보:

- 기본금리와 최고금리 차이
- 신규가입/첫거래 조건
- 급여이체, 주거래통장, 카드실적, 자동이체, 마케팅 동의 조건
- 상세 `금리 안내` 원문과 우대금리별 달성 조건
- 나이, 소득, 첫가입, 납입한도, 우대금리 적용 기간
- 만기별 금리 차이
- 최대 가입금액 또는 월 납입 한도
- 앱/비대면 가입 가능 여부
- 예금자보호 여부 확인 필요 상태

운영 명령:

```bash
npm run daily-naver-products
npm run review-products -- list
npm run review-products -- approve --bank "은행명" --name "상품명 12개월" --type deposit --official-url "공식URL"
```

### 6.4 데이터 품질 관리

필수 관리 항목:

- 업데이트 날짜 표시
- 최고금리와 기본금리 분리
- 우대조건을 사람이 이해할 수 있게 요약

- 제휴 링크가 있다면 경제적 이해관계 표시

## 7. 최적 계산 파이프라인

### 7.1 계산 목표

사용자 조건 안에서 세후 예상 이자를 최대화한다.

단, 단순히 금리만 높은 상품을 고르는 것이 아니라 다음 조건을 함께 고려한다.

- 사용 예정 시점
- 중도 인출 필요성
- 상품별 가입 한도
- 월 납입 한도
- 예금자보호 한도
- 우대금리 달성 가능성
- 상품 가입 난이도
- 만기 분산
- 세후 이자

### 7.2 기본 계산 흐름

1. 사용자 입력값을 받는다.
2. 사용 기간과 유동성 필요 여부를 기준으로 상품 후보를 필터링한다.
3. 우대조건 가능 여부에 따라 적용 금리를 계산한다.
4. 세후 이자를 계산한다.
5. 목돈과 매월 저축액을 상품별 한도 안에서 배분한다.
6. 예상 이자와 추가 이익을 계산한다.
7. 추천 결과를 사람이 이해하기 쉬운 형태로 설명한다.

### 7.3 추천 로직 초기 버전

초기에는 복잡한 최적화 엔진보다 규칙 기반으로 시작한다.

예시 규칙:

- 3개월 이내 사용할 돈은 파킹통장 우선
- 6개월 이상 묶을 수 있는 목돈은 정기예금 후보에 포함
- 매월 납입하는 돈은 적금 후보에 포함
- 중도 인출 가능성이 높으면 파킹통장 비중 확대
- 예금자보호 한도를 넘으면 금융사 분산 추천
- 우대조건을 달성할 수 없는 상품은 기본금리 기준으로 계산
- 세후 이자 차이가 작으면 간편가입 상품을 우선 노출

### 7.4 고도화 방향

이후에는 다음 방식으로 고도화할 수 있다.

- 선형계획법 기반 최적화
- 사용자 유형별 가중치 적용
- 목적자금별 만기 매칭
- 세후 수익률 기준 정렬
- 만기일 분산 알고리즘
- 금리 변동 시 리밸런싱 추천

## 8. 수익화 전략

### 8.1 중요한 전제

저축 최적화 서비스는 사용 빈도가 높지 않다. 사용자는 보통 매월 접속하지 않고, 특정 이벤트가 있을 때 접속한다.

대표 이벤트:

- 월급 또는 상여금 입금
- 예금/적금 만기
- 목돈 발생
- 금리 변동
- 목적자금 계획

따라서 처음부터 월구독에 의존하면 수익성이 약할 수 있다.

### 8.2 추천 수익모델

초기 추천 구조:

> 무료 계산기 → 알림 신청 → 1회 상세 리포트 판매 → 만기/금리 알림 → 재방문 → 금융사 제휴

수익모델:

- 무료 계산기
- 1회 최적 포트폴리오 리포트
- 만기 알림
- 금리 변동 알림
- 연간 자산 캘린더
- 금융사 광고
- 금융사 제휴 링크
- B2B 데이터/계산기 위젯 제공

### 8.3 가격 제안

초기 테스트 가격:

- 1회 상세 리포트: 4,900원~9,900원
- 추천 시작 가격: 6,900원
- 월 구독: 1,900원~3,900원
- 추천 시작 가격: 월 2,900원
- 연간 구독: 19,000원~29,000원
- 추천 시작 가격: 연 24,000원

다만 월구독은 핵심 모델이 아니라 보조 모델로 두는 것이 좋다.

### 8.4 수익 추정

단순 MAU 기준이 아니라, 실제 계산기를 쓰는 이벤트성 활성 사용자를 기준으로 봐야 한다.

예시:

- 전체 사용자 10,000명
- 매월 실제 계산기 사용률 10%~20%
- 월 계산기 사용자 1,000명~2,000명
- 리포트 결제 전환율 3%~5%
- 리포트 가격 6,900원

예상 리포트 매출:

- 1,000명 x 3% x 6,900원 = 약 207,000원
- 2,000명 x 5% x 6,900원 = 약 690,000원

즉, 사용자 10,000명이어도 리포트 판매만으로는 월 20만~70만 원 수준일 수 있다.

수익을 키우려면 다음이 필요하다.

- 재방문 트리거 확보
- 만기 알림
- 금리 변동 알림
- 금융사 제휴
- 카드/절약/투자/목적자금 관리로 확장

## 9. 규제 및 신뢰 이슈

금융상품 비교, 추천, 가입 연결은 단순 콘텐츠보다 규제 리스크가 높다.

초기에는 다음 원칙을 지킨다.

- 투자 조언처럼 보이는 표현을 피한다.
- 예금, 적금, 파킹통장 중심의 정보 제공으로 시작한다.
- "추천"보다는 "계산 결과", "조건별 정렬", "예상 이자 비교" 표현을 사용한다.
- 제휴 링크가 있으면 경제적 이해관계를 명확히 표시한다.
- 광고 상품과 계산 결과를 분리한다.
- 순위가 광고비 때문에 바뀌지 않도록 한다.
- 금리와 조건의 업데이트 날짜를 명확히 표시한다.
- 실제 가입 전 금융사 공식 페이지에서 조건을 확인하라고 안내한다.

추후 금융사 제휴나 가입 중개를 본격화하려면 금융소비자보호법, 금융상품판매대리/중개 관련 규제를 별도로 검토해야 한다.

## 10. 확장 로드맵

### 10.1 1단계: 저축 최적화 계산기

목표:

- 빠르게 웹페이지를 만든다.
- 사용자가 자신의 월 저축액과 목돈을 입력하게 한다.
- 결과로 예금, 적금, 파킹통장 배분안을 보여준다.

핵심:

- 무료 계산기
- 세후 이자 계산
- 추가 이익 표시
- 공식 가입 링크

### 10.2 2단계: 저장과 알림

목표:

- 이벤트성 서비스를 재방문 서비스로 바꾼다.

기능:

- 만기일 저장
- 금리 변동 알림
- 이메일/카카오 알림
- 내 포트폴리오 저장
- 리밸런싱 추천

### 10.3 3단계: 목적자금 플랜

목표:

- 단순 저축 계산기에서 개인 목적자금 관리로 확장한다.

기능:

- 전세자금 플랜
- 결혼자금 플랜
- 자동차 구매자금 플랜
- 여행/유학자금 플랜
- 비상금 플랜

### 10.4 4단계: 개인 돈 관리 이벤트 허브

목표:

- 저축, 절약, 카드 혜택, 투자 기초 정보를 연결한다.

확장 후보:

- 월급 자동 배분 계산기
- 카드 혜택 추천
- 생활비 절약 체크
- 생필품 최저가 비교
- ISA/연금저축 기초 가이드
- 초보 투자자용 ETF 비교

## 11. 가장 빠른 실행 순서

1. 상품 데이터 스키마를 정의한다.
2. Google Sheets 또는 간단한 DB에 예금/적금/파킹통장 데이터를 넣는다.
3. 사용자 입력 폼을 만든다.
4. 세후 이자 계산 로직을 만든다.
5. 규칙 기반 추천 로직을 만든다.
6. 결과 화면을 만든다.
7. 공식 가입 링크를 붙인다.
8. 이메일 알림 신청을 붙인다.
9. 상세 리포트 결제 버튼을 테스트한다.
10. 실제 사용자 반응을 보고 가격과 기능을 조정한다.

## 12. 초기 성공 지표

초기에는 매출보다 반응을 봐야 한다.

확인할 지표:

- 방문자 중 계산 완료율
- 계산 완료 후 결과 체류 시간
- 공식 가입 링크 클릭률
- 이메일/알림 신청률
- 상세 리포트 구매 전환율
- 사용자가 입력하는 평균 목돈 규모
- 가장 많이 선택하는 사용 기간
- 가장 많이 선택하는 목적

초기 목표:

- 계산 완료율 30% 이상
- 결과 페이지 링크 클릭률 10% 이상
- 알림 신청률 5% 이상
- 리포트 구매 전환율 1%~3%

## 13. 결론

이 아이디어는 "매일 쓰는 앱"이 아니라 "돈을 배치해야 하는 순간에 쓰는 계산기"로 시작하는 것이 맞다.

초기 핵심은 다음 세 가지다.

- 정확한 금융상품 데이터
- 사용자의 돈과 기간에 맞춘 계산
- 숫자로 보이는 추가 이익

가장 빠른 MVP는 다음 문장으로 설명할 수 있어야 한다.

> 월 저축액과 목돈을 입력하면, 지금 가능한 예금, 적금, 파킹통장 중 가장 효율적인 배분을 세후 이자 기준으로 계산해주는 웹페이지.

장기적으로는 만기 알림, 금리 변동 알림, 목적자금 플랜, 카드/절약/투자 정보까지 확장해 **개인 돈 관리 이벤트 허브**로 키우는 것이 좋다.
